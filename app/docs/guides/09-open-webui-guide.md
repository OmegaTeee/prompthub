# Open WebUI — Local Chat Interface for PromptHub

## Overview

Open WebUI is a chat interface that connects to PromptHub's OpenAI-compatible API. All conversations stay local — traffic flows from Open WebUI through PromptHub to Ollama on your Mac.

```
Open WebUI (localhost:3000)
    → PromptHub /v1/ proxy (localhost:9090)
        → Ollama (localhost:11434)
```

### What You Get

- Chat UI with conversation history, folders, and search
- Model selector for all your Ollama models (gemma3:4b, gemma3:27b, qwen3:14b, etc.)
- RAG — upload documents or connect your Obsidian vault for retrieval-augmented answers
- MCP tool access via PromptHub (web search, file operations, note search)
- All requests logged in the PromptHub dashboard (`localhost:9090/dashboard`)

## Installation

Open WebUI is installed via `uv` (the Python package runner). It is **not** a standalone app — you launch it from the terminal each time.

### Where Things Live

| Component          | Location                                                   |
| ------------------ | ---------------------------------------------------------- |
| Open WebUI binary  | Managed by `uvx` (cached in `~/.cache/uv/`)                |
| Database & uploads | `~/.open-webui/`                                           |
| Secret key         | `~/.webui_secret_key`                                      |
| PromptHub API key  | `app/configs/api-keys.json` (`sk-prompthub-openwebui-001`) |
| PromptHub config   | `~/.prompthub/open-webui.json` (generated by CLI)          |
| Launch scripts     | `app/scripts/open-webui/start.sh`, `stop.sh`               |
| LaunchAgent        | `app/scripts/launch-agents/com.prompthub.openwebui.plist`  |
| Logs               | `~/.prompthub/open-webui.log`                              |

### Data Directory Contents

```
~/.open-webui/
├── webui.db        # SQLite database (users, chats, settings)
├── cache/          # Model cache and temp files
├── uploads/        # Uploaded documents (for RAG)
└── vector_db/      # Embeddings for knowledge base / RAG
```

## Launching Open WebUI

### Prerequisites

1. **uv** must be installed (`~/.local/bin/uv`)
2. **PromptHub** must be running on port 9090
3. **Ollama** must be running on port 11434

### Start Command

```bash
cd ~ && DATA_DIR=~/.open-webui \
  ENABLE_OPENAI_API=true \
  OPENAI_API_BASE_URL=http://localhost:9090/v1 \
  OPENAI_API_KEY=sk-prompthub-openwebui-001 \
  uvx --python 3.11 "open-webui>=0.8,<0.8.10" serve --port 3000
```

Then open http://localhost:3000 in your browser.

### What the Environment Variables Do

| Variable | Value | Purpose |
|----------|-------|---------|
| `DATA_DIR` | `~/.open-webui` | Persistent storage directory |
| `ENABLE_OPENAI_API` | `true` | Enable the OpenAI-compatible backend |
| `OPENAI_API_BASE_URL` | `http://localhost:9090/v1` | Point to PromptHub (not Ollama directly) |
| `OPENAI_API_KEY` | `sk-prompthub-openwebui-001` | Bearer token registered in PromptHub |

### Stopping Open WebUI

Press `Ctrl+C` in the terminal where it's running, or use the stop script (see below).

### Launch Scripts

Ready-made shell scripts are provided in `app/scripts/open-webui/` so you do not have to remember the full `uvx` command and environment variables.

**Start Open WebUI:**

```bash
./app/scripts/open-webui/start.sh
```

The start script:

- Reads settings from `~/.prompthub/open-webui.json` (generated by the CLI -- see below)
- Checks that Ollama (port 11434) and PromptHub (port 9090) are running, and warns if either is down
- Detects port conflicts before launching
- Logs output to `~/.prompthub/open-webui.log`

**Stop Open WebUI:**

```bash
./app/scripts/open-webui/stop.sh
```

**Override the port:**

```bash
OWUI_PORT=8080 ./app/scripts/open-webui/start.sh
OWUI_PORT=8080 ./app/scripts/open-webui/stop.sh
```

### Auto-Start on Login (LaunchAgent)

A macOS LaunchAgent plist is available at `app/scripts/launch-agents/com.prompthub.openwebui.plist`. To install it:

```bash
# Copy plist to LaunchAgents
cp app/scripts/launch-agents/com.prompthub.openwebui.plist \
   ~/Library/LaunchAgents/

# Load (does not start immediately — RunAtLoad is false)
launchctl load ~/Library/LaunchAgents/com.prompthub.openwebui.plist

# Start manually
launchctl start com.prompthub.openwebui

# View logs
tail -f ~/.prompthub/open-webui-launchd.log
```

To uninstall:

```bash
launchctl unload ~/Library/LaunchAgents/com.prompthub.openwebui.plist
rm ~/Library/LaunchAgents/com.prompthub.openwebui.plist
```

## CLI Management

Open WebUI is a first-class PromptHub client. The MCP Config Manager CLI can generate, install, and validate its connection settings, just like it does for Claude Desktop, VS Code, and Raycast.

### Generate Configuration

```bash
cd app && python -m cli generate open-webui
```

This prints the connection settings JSON to stdout. The output looks like:

```json
{
  "open_webui": {
    "api_base_url": "http://127.0.0.1:9090/v1",
    "mcp_endpoint": "http://127.0.0.1:9090/mcp-direct/mcp",
    "api_key": "sk-prompthub-openwebui-001",
    "port": 3000,
    "data_dir": "~/.open-webui"
  }
}
```

### Install Configuration

```bash
cd app && python -m cli install open-webui
```

This writes the config to `~/.prompthub/open-webui.json`. The launch scripts read this file automatically on startup.

### Validate Configuration

```bash
cd app && python -m cli validate open-webui
```

Validation checks:

- The config file exists at `~/.prompthub/open-webui.json`
- The `open_webui` section is present
- `api_base_url` uses `127.0.0.1` (not `localhost`, which can cause IPv6 issues)
- `api_key` is present and matches a key in `configs/api-keys.json`

### List All Clients

```bash
cd app && python -m cli list
```

Open WebUI now appears alongside Claude Desktop, VS Code, Cursor, and Raycast in the client list.

### Full Stack Diagnosis

```bash
cd app && python -m cli diagnose
```

Runs health checks against Ollama, PromptHub, and all configured clients.

## First-Time Setup

### 1. Create Your Account

The first user to register becomes the admin. Open http://localhost:3000 and create an account with any email/password (stored locally in `~/.open-webui/webui.db`).

### 2. Verify Connection

After logging in, your Ollama models should appear in the model selector dropdown at the top of the chat page. If they don't:

1. Click your profile icon (bottom-left) → **Admin Panel**
2. Go to **Settings → Connections**
3. Under **OpenAI API**, verify:
   - **API Base URL:** `http://localhost:9090/v1`
   - **API Key:** `sk-prompthub-openwebui-001`
4. Click **Save**

### 3. Choose a Model

| Model | Best For | Speed |
|-------|----------|-------|
| `gemma3:4b` | Quick questions, casual chat | Fast |
| `gemma3:27b` | Detailed answers, writing help | Medium |
| `qwen3:14b` | Reasoning, analysis | Medium |
| `qwen3-coder:30b` | Code-related questions | Slower |

## Using MCP Tools

PromptHub manages MCP servers (web search, file operations, Obsidian notes, etc.). You can access these from Open WebUI in two ways:

### Option A: Native MCP (Streamable HTTP)

Open WebUI supports MCP via Streamable HTTP natively (since v0.6.31):

1. Go to **Admin Panel** (click profile icon bottom-left) → **Settings → External Tools**
2. Click **+ (Add Server)**
3. Set **Type** to **"MCP (Streamable HTTP)"** (not OpenAPI)
4. Set **Server URL** to: `http://localhost:9090/mcp-direct/mcp`
5. Leave authentication as **None** (local traffic, no token needed)
6. Click **Save**

All PromptHub tools (web search, file operations, sequential-thinking, Obsidian notes, etc.) are discovered automatically.

> **Note:** PromptHub's gateway runs in stateless mode — no session handshake is needed. Each request is independent, which is compatible with Open WebUI's MCP client.

### Option B: Custom Tool Functions

1. Go to **Workspace → Tools → Create Tool**
2. Paste a Python tool class that calls PromptHub's MCP proxy
3. Enable the tool on any chat via the **+** button next to the message input

Example tool function for web search and file operations:

```python
"""
title: PromptHub MCP Tools
description: Call MCP tools through PromptHub router
version: 0.1.0
"""

import httpx
import json


class Tools:
    def __init__(self):
        self.base_url = "http://localhost:9090"

    async def search_web(self, query: str) -> str:
        """Search the web using DuckDuckGo."""
        return await self._call_tool(
            "duckduckgo", "search", {"query": query, "count": 5}
        )

    async def read_file(self, path: str) -> str:
        """Read a file from the local filesystem."""
        return await self._call_tool(
            "desktop-commander", "read_file", {"path": path}
        )

    async def list_directory(self, path: str) -> str:
        """List files in a directory."""
        return await self._call_tool(
            "desktop-commander", "list_directory", {"path": path}
        )

    async def _call_tool(self, server: str, tool: str, args: dict) -> str:
        async with httpx.AsyncClient(timeout=30.0) as client:
            resp = await client.post(
                f"{self.base_url}/mcp/{server}/tools/call",
                headers={
                    "Content-Type": "application/json",
                    "X-Client-Name": "openwebui",
                },
                json={
                    "jsonrpc": "2.0",
                    "method": "tools/call",
                    "params": {"name": tool, "arguments": args},
                    "id": 1,
                },
            )
            data = resp.json()
            if "error" in data:
                return f"Error: {data['error']['message']}"
            result = data.get("result", {})
            if isinstance(result, dict) and "text" in result:
                return result["text"]
            return json.dumps(result, indent=2)
```

## Prompt Enhancement

Open WebUI has automatic prompt enhancement enabled by default. When you send a message in Open WebUI, PromptHub rewrites the prompt to be clearer and more specific before forwarding it to Ollama.

**Configuration details:**

| Setting         | Value                          |
| --------------- | ------------------------------ |
| Model           | `gemma3:4b`                    |
| Privacy level   | `local_only` (never leaves your Mac) |
| API key         | `sk-prompthub-openwebui-001`   |
| Enhancement     | Enabled (`"enhance": true`)    |

Enhancement is configured in two places:

- **`configs/api-keys.json`** -- the API key entry has `"enhance": true`, which tells the `/v1/chat/completions` proxy to rewrite prompts before sending them to Ollama.
- **`configs/enhancement-rules.json`** -- the `open-webui` client section defines the model, system prompt, temperature, and privacy level.

To disable enhancement, set `"enhance": false` for the `sk-prompthub-openwebui-001` key in `configs/api-keys.json` and restart the router.

## Monitoring

All Open WebUI requests flow through PromptHub. You can monitor them at:

- **Dashboard:** http://localhost:9090/dashboard (activity feed, circuit breakers, Ollama status, Open WebUI panel)
- **Activity log:** `GET http://localhost:9090/audit/activity?client_id=openwebui`
- **Health check:** `curl http://localhost:9090/health`

### Dashboard Panel

The PromptHub dashboard includes a dedicated Open WebUI panel that shows:

- **Connection status** -- a green or red badge indicating whether Open WebUI is responding on its configured port (default 3000)
- **API base URL** and **MCP endpoint** -- the URLs Open WebUI uses to reach PromptHub
- **Port** -- the port Open WebUI is running on
- **Recent request count** -- how many requests Open WebUI has sent recently
- **Quick link** -- a button to open Open WebUI in your browser

The panel auto-refreshes every 15 seconds via HTMX polling.

### Configuration API Endpoint

Connection settings are also available as JSON from the router itself:

```bash
curl http://localhost:9090/configs/open-webui
```

This returns the same structure as `python -m cli generate open-webui`. Useful for scripting or verifying the running configuration.

## Troubleshooting

### No models showing up

```bash
# Verify PromptHub is running
curl http://localhost:9090/health

# Verify the API key works
curl -H "Authorization: Bearer sk-prompthub-openwebui-001" \
  http://localhost:9090/v1/models
```

If the key returns 401, reload keys:
```bash
curl -X POST http://localhost:9090/v1/api-keys/reload
```

### Chat returns errors

```bash
# Check if Ollama is running
curl http://localhost:11434/api/tags

# Check PromptHub circuit breakers (on dashboard)
open http://localhost:9090/dashboard
```

### Port 3000 already in use

```bash
# Find what's using port 3000
lsof -ti tcp:3000

# Kill it, or use a different port
uvx --python 3.11 "open-webui>=0.8,<0.8.10" serve --port 3001
```

### Version update available but broken

Open WebUI v0.8.10 has a broken dependency (`ddgs==9.11.2`). Stay on `<0.8.10` until it's fixed upstream. Check periodically:

```bash
uvx --python 3.11 open-webui@latest serve --port 3000
```

If it fails with a dependency error, revert to the pinned version.
