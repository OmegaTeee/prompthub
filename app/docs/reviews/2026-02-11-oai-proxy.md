# OpenAI-Compatible API Proxy for PromptHub Router

## Context

Desktop apps like Cursor, Raycast extensions, and Obsidian plugins only support OpenAI-compatible API endpoints (`/v1/chat/completions`). Currently, they must connect directly to Ollama at `localhost:11434/v1/`, bypassing the router's enhancement pipeline, caching, circuit breaking, and audit logging. This plan adds `/v1/` endpoints to the router so desktop apps point to `localhost:9090/v1/` and transparently get all router features.

## Request Flow

```
Desktop App → POST localhost:9090/v1/chat/completions
  → Bearer token validation → resolve client_name
  → Enhancement: last user message enhanced via EnhancementService
  → Circuit breaker check (fail fast if Ollama is down)
  → Forward to Ollama (localhost:11434/v1/chat/completions)
  → Stream SSE or return JSON → back to app
```

**1. `app/configs/api-keys.json`** — Token-to-client mapping

```json

      "client_name": "cursor",
      "enhance": true,
      "description": "Cursor IDE"

    "sk-prompthub-obsidian-xxx": {
```

- `client_name` maps to entries in `enhancement-rules.json`

- Prefix convention `sk-prompthub-{client}-{random}` for log identification

### Phase 2: Core Router (depends on Phase 1)

**5. `app/router/openai_compat/streaming.py`** — SSE relay

- `stream_ollama_response(ollama_base_url, payload, timeout)` async generator
- Ollama already produces OpenAI-compatible SSE chunks — this is a relay, not a transformation
- Uses `httpx.AsyncClient.stream()` for async chunk iteration
- Appends `data: [DONE]\n\n` sentinel

**6. `app/router/openai_compat/router.py`** — Factory function with endpoints

- `create_openai_compat_router(enhancement_service, circuit_breakers, api_key_manager, ollama_base_url, ollama_timeout) -> APIRouter`
- Follows exact pattern from `app/router/dashboard/router.py:29-54`

Endpoints:

|Endpoint|Auth|Description|
|---|---|---|
|`POST /v1/chat/completions`|Bearer|Chat completion with optional enhancement + streaming|
|`GET /v1/models`|Bearer|List available Ollama models|
|`POST /v1/api-keys/reload`|None (admin)|Hot-reload API keys config|

**`POST /v1/chat/completions` flow:**

1. Validate bearer token via `authenticate` dependency → get `ApiKeyConfig`
2. Check circuit breaker for `"ollama"` → 503 if OPEN
3. If `api_key.enhance` is true: find last user message, call `enhancement_service.enhance(prompt, client_name)`, replace content if enhanced
4. Build Ollama payload from request body
5. **If streaming:** Use `stream_ollama_response()` → `StreamingResponse(media_type="text/event-stream")`
6. **If non-streaming:** Use existing `OllamaOpenAIClient.chat_completion()` from `ollama_openai.py:174` → return `response.model_dump()`
7. Record success/failure to circuit breaker, audit all outcomes

## Key Design Decisions

1. **Only last user message enhanced** — System messages and conversation history are never modified
2. **Enhancement before streaming** — Enhance first (~100-300ms), then stream the Ollama response
3. **401 if no token** — No anonymous access; token determines which enhancement rules apply
4. **Separate circuit breaker** — Use `"ollama-proxy"` to avoid coupling with the enhancement service's breaker
5. **Reuse `OllamaOpenAIClient`** for non-streaming — Already has retry logic, error handling, response parsing
6. **Relay SSE for streaming** — Ollama produces OpenAI-compatible SSE; no transformation needed

### How Desktop Apps Connect

Configure each app's OpenAI settings:

- **Base URL**: `http://localhost:9090/v1`
- **API Key**: `sk-prompthub-cursor-dev001` (or the appropriate key from `api-keys.json`)
- **Model**: Any Ollama model name (e.g., `llama3.2:latest`, `deepseek-r1:latest`)

> ★ **The enhancement pipeline is invisible to desktop apps.** From Cursor's perspective, it's just talking to an OpenAI-compatible API. But behind the scenes, the router enhances the last user message using per-client rules (Cursor gets `qwen3-coder`, Obsidian gets pass-through), caches enhanced prompts (SHA-256 keyed LRU), checks the circuit breaker, and logs everything to the audit trail. The app never knows — it just gets better responses with resilience for free.
