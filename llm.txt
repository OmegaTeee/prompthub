# PromptHub — Project Context for LLM Assistants

> This file is for external LLM tools (Perplexity, etc.) that read project files
> via Desktop Commander or similar MCP tools. It provides a self-contained snapshot
> of the project so you can give informed recommendations without reading the full codebase.
>
> Last updated: 2026-04-05

## What this project is

PromptHub is a local MCP (Model Context Protocol) router for macOS. It runs on
localhost:9090 and provides:

- **MCP server multiplexing** — One endpoint proxies 9 MCP servers (context7,
  desktop-commander, sequential-thinking, memory, duckduckgo, obsidian-mcp-tools,
  chrome-devtools-mcp, perplexity-comet, browsermcp) to any client
- **Prompt enhancement** — Local LLM rewrites prompts per-client before they reach
  the AI (different system prompts for code editors vs research tools vs image gen)
- **OpenAI-compatible proxy** — `/v1/chat/completions` with bearer auth, so any
  OpenAI-compatible client can use local models through PromptHub
- **Resilience** — Circuit breakers, L1/L2 cache, cloud fallback via OpenRouter

## Tech stack

- **Backend**: Python 3.14, FastAPI, Pydantic, uvicorn, httpx, aiosqlite
- **MCP bridge**: Node.js stdio bridge (`mcps/prompthub-bridge.js`) aggregating
  all servers behind a single process
- **Local LLM**: LM Studio (port 1234), OpenAI-compatible API
- **Dashboard**: HTMX + Jinja2 templates at `/dashboard`
- **Persistent storage**: SQLite databases under `~/.prompthub/`

## Key design decisions

- **Privacy boundary**: `PrivacyLevel` enum (local_only, free_ok, any) controls
  whether prompts leave localhost. Most clients default to local_only.
- **Cloud fallback**: When LM Studio is down, free_ok/any clients fall back to
  OpenRouter free tier. local_only clients never use cloud.
- **Single enhancement model**: All clients use the same model (`qwen3-4b-instruct-2507`)
  with per-client system prompts and temperature settings.
- **Schema minification**: Bridge strips verbose fields from tool schemas before
  sending to LLM clients, reducing context by ~67%.
- **Token budget**: Enhancement input capped at 4,096 tokens to prevent wasting
  context on prompt rewrites.
