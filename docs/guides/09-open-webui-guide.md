# Open WebUI -- Local Chat Interface for PromptHub

## Overview

Open WebUI gives you a browser-based chat interface that connects to PromptHub. All conversations stay on your Mac. Think of it like a private ChatGPT that runs entirely on your own computer.

Here is how traffic flows:

```
Open WebUI (localhost:3000)
    → PromptHub /v1/ proxy (localhost:9090)
        → Ollama (localhost:11434)
```

### What You Get

- A chat UI with conversation history, folders, and search.
- A model selector for all your Ollama models (gemma3:4b, gemma3:27b, qwen3:14b, and more).
- RAG (Retrieval-Augmented Generation): upload documents or connect your Obsidian vault so the AI can reference them when answering. RAG is like giving the AI a reference bookshelf it can check before responding.
- MCP tool access through PromptHub (web search, file operations, note search).
- All requests logged in the PromptHub dashboard at `localhost:9090/dashboard`.

**Key points:**
- Open WebUI is a local chat interface that runs in your browser.
- All data stays on your Mac. Nothing leaves your network.
- PromptHub sits between Open WebUI and Ollama, adding enhancement and tool access.

## Installation

Open WebUI is installed using `uv`, a Python package runner. It is not a standalone app. You launch it from the terminal each time.

### Where Things Live

| Component          | Location                                                   |
| ------------------ | ---------------------------------------------------------- |
| Open WebUI binary  | Managed by `uvx` (cached in `~/.cache/uv/`)                |
| Database and uploads | `~/.open-webui/`                                           |
| Secret key         | `~/.webui_secret_key`                                      |
| PromptHub API key  | `app/configs/api-keys.json` (`sk-prompthub-openwebui-001`) |
| PromptHub config   | `~/.prompthub/open-webui.json` (generated by CLI)          |
| Launch scripts     | `scripts/open-webui/start.sh`, `stop.sh`               |
| LaunchAgent        | `scripts/launch-agents/com.prompthub.openwebui.plist`  |
| Logs               | `logs/openwebui.log`                                       |

### Data Directory Contents

```
~/.open-webui/
├── webui.db        # SQLite database (users, chats, settings)
├── cache/          # Model cache and temp files
├── uploads/        # Uploaded documents (for RAG)
└── vector_db/      # Embeddings for knowledge base / RAG
```

**Key points:**
- Open WebUI stores all its data in `~/.open-webui/`.
- The connection config lives at `~/.prompthub/open-webui.json`.
- You need `uv` installed to run Open WebUI.

## Launching Open WebUI

### Prerequisites

Before you start, make sure these three things are running:

1. **uv** must be installed (at `~/.local/bin/uv`).
2. **PromptHub** must be running on port 9090.
3. **Ollama** must be running on port 11434.

### Start Command

Run this in your terminal:

```bash
cd ~ && DATA_DIR=~/.open-webui \
  ENABLE_OPENAI_API=true \
  OPENAI_API_BASE_URL=http://127.0.0.1:9090/v1 \
  OPENAI_API_KEY=sk-prompthub-openwebui-001 \
  uvx --python 3.11 "open-webui>=0.8,<0.8.10" serve --port 3000
```

Then open http://localhost:3000 in your browser.

### What the Environment Variables Do

| Variable | Value | Purpose |
|----------|-------|---------|
| `DATA_DIR` | `~/.open-webui` | Where Open WebUI stores its data |
| `ENABLE_OPENAI_API` | `true` | Turns on the OpenAI-compatible backend |
| `OPENAI_API_BASE_URL` | `http://localhost:9090/v1` | Points to PromptHub (not Ollama directly) |
| `OPENAI_API_KEY` | `sk-prompthub-openwebui-001` | Bearer token registered in PromptHub |

### Stopping Open WebUI

Press `Ctrl+C` in the terminal where it is running, or use the stop script (described below).

### Launch Scripts

Ready-made shell scripts are provided in `scripts/open-webui/` so you do not have to remember the full command.

**Start Open WebUI:**

```bash
./scripts/open-webui/start.sh
```

The start script does the following:

1. Reads settings from `~/.prompthub/open-webui.json` (generated by the CLI).
2. Checks that Ollama (port 11434) and PromptHub (port 9090) are running. It warns you if either is down.
3. Detects port conflicts before launching.
4. Logs output to `logs/openwebui.log`.

**Stop Open WebUI:**

```bash
./scripts/open-webui/stop.sh
```

**Override the port:**

```bash
OWUI_PORT=8080 ./scripts/open-webui/start.sh
OWUI_PORT=8080 ./scripts/open-webui/stop.sh
```

### Auto-Start on Login (LaunchAgent)

You can set Open WebUI to launch automatically when you log in to your Mac. A LaunchAgent is a macOS feature that starts background services at login, like setting an alarm clock for your apps.

A plist file is available at `scripts/launch-agents/com.prompthub.openwebui.plist`.

To install it:

```bash
# Symlink plist into LaunchAgents (edits to the repo copy take effect on next load)
ln -sf ~/.local/share/prompthub/scripts/launch-agents/com.prompthub.openwebui.plist \
   ~/Library/LaunchAgents/

# Load (does not start immediately — RunAtLoad is false)
launchctl load ~/Library/LaunchAgents/com.prompthub.openwebui.plist

# Start manually
launchctl start com.prompthub.openwebui

# View logs
tail -f ~/prompthub/logs/openwebui-stderr.log
```

To uninstall:

```bash
launchctl unload ~/Library/LaunchAgents/com.prompthub.openwebui.plist
rm ~/Library/LaunchAgents/com.prompthub.openwebui.plist
```

**Key points:**
- Use the start script (`start.sh`) for the easiest launch experience.
- The LaunchAgent lets Open WebUI start automatically on login.
- Press `Ctrl+C` or run `stop.sh` to shut it down.

## CLI Management

Open WebUI is a first-class PromptHub client. The MCP Config Manager CLI can generate, install, and validate its connection settings, the same way it does for Claude Desktop, VS Code, and Raycast.

### Generate Configuration

```bash
cd app && python -m cli generate open-webui
```

This prints the connection settings JSON to stdout:

```json
{
  "open_webui": {
    "api_base_url": "http://127.0.0.1:9090/v1",
    "mcp_endpoint": "http://127.0.0.1:9090/mcp-direct/mcp",
    "api_key": "sk-prompthub-openwebui-001",
    "port": 3000,
    "data_dir": "~/.open-webui"
  }
}
```

### Install Configuration

```bash
cd app && python -m cli install open-webui
```

This writes the config to `~/.prompthub/open-webui.json`. The launch scripts read this file automatically on startup.

### Validate Configuration

```bash
cd app && python -m cli validate open-webui
```

Validation checks the following:

1. The config file exists at `~/.prompthub/open-webui.json`.
2. The `open_webui` section is present.
3. `api_base_url` uses `127.0.0.1` (not `localhost`, which can cause IPv6 issues).
4. `api_key` is present and matches a key in `configs/api-keys.json`.

### List All Clients

```bash
cd app && python -m cli list
```

Open WebUI appears alongside Claude Desktop, VS Code, Cursor, and Raycast in the client list.

### Full Stack Diagnosis

```bash
cd app && python -m cli diagnose
```

This runs health checks against Ollama, PromptHub, and all configured clients.

**Key points:**
- Use `generate` to see the config, `install` to write it, and `validate` to verify it.
- Always use `127.0.0.1` instead of `localhost` to avoid IPv6 problems.
- Run `diagnose` to check the health of your entire stack at once.

## First-Time Setup

### 1. Create Your Account

The first person to register becomes the admin. Open http://localhost:3000 and create an account with any email and password. This account is stored locally in `~/.open-webui/webui.db`.

### 2. Verify the Connection

After logging in, your Ollama models should appear in the model selector dropdown at the top of the chat page. If they do not show up:

1. Click your profile icon (bottom-left) and select **Admin Panel**.
2. Go to **Settings** then **Connections**.
3. Under **OpenAI API**, verify these values:
   - **API Base URL:** `http://localhost:9090/v1`
   - **API Key:** `sk-prompthub-openwebui-001`
4. Click **Save**.

### 3. Choose a Model

| Model | Best For | Speed |
|-------|----------|-------|
| `gemma3:4b` | Quick questions, casual chat | Fast |
| `gemma3:27b` | Detailed answers, writing help | Medium |
| `qwen3:14b` | Reasoning, analysis | Medium |
| `qwen3-coder:30b` | Code-related questions | Slower |

Pick a model from the dropdown at the top of the chat page. Start with `gemma3:4b` for speed, or `gemma3:27b` for quality.

**Key points:**
- The first user to register becomes the admin.
- Check **Settings > Connections** if models do not appear.
- Start with `gemma3:4b` for fast responses and upgrade as needed.

## Using MCP Tools

PromptHub manages MCP servers that provide capabilities like web search, file operations, and Obsidian note access. You can use these tools from Open WebUI in two ways.

### Option A: Native MCP (Streamable HTTP)

Open WebUI supports MCP natively through Streamable HTTP (since version 0.6.31). This is the recommended approach.

1. Click your profile icon (bottom-left) and select **Admin Panel**.
2. Go to **Settings** then **External Tools**.
3. Click **+ (Add Server)**.
4. Set **Type** to **"MCP (Streamable HTTP)"** (not OpenAPI).
5. Set **Server URL** to: `http://127.0.0.1:9090/mcp-direct/mcp`
6. Leave authentication as **None** (local traffic does not need a token).
7. Click **Save**.

All PromptHub tools (web search, file operations, sequential-thinking, Obsidian notes, and more) are discovered automatically.

> **Note:** Open WebUI handles the Streamable HTTP session handshake on its own. The gateway exposes **58 tools** across all running MCP servers. Tools are namespaced by server name (for example, `context7_resolve-library-id`).

### Option B: Custom Tool Functions

If you want more control, you can write Python tool functions that call PromptHub's MCP proxy directly.

1. Go to **Workspace** then **Tools** then **Create Tool**.
2. Paste the Python tool class below.
3. Enable the tool on any chat by clicking the **+** button next to the message input.

Example tool function for web search and file operations:

```python
"""
title: PromptHub MCP Tools
description: Call MCP tools through PromptHub router
version: 0.1.0
"""

import httpx
import json


class Tools:
    def __init__(self):
        self.base_url = "http://localhost:9090"

    async def search_web(self, query: str) -> str:
        """Search the web using DuckDuckGo."""
        return await self._call_tool(
            "duckduckgo", "search", {"query": query, "count": 5}
        )

    async def read_file(self, path: str) -> str:
        """Read a file from the local filesystem."""
        return await self._call_tool(
            "desktop-commander", "read_file", {"path": path}
        )

    async def list_directory(self, path: str) -> str:
        """List files in a directory."""
        return await self._call_tool(
            "desktop-commander", "list_directory", {"path": path}
        )

    async def _call_tool(self, server: str, tool: str, args: dict) -> str:
        async with httpx.AsyncClient(timeout=30.0) as client:
            resp = await client.post(
                f"{self.base_url}/mcp/{server}/tools/call",
                headers={
                    "Content-Type": "application/json",
                    "X-Client-Name": "openwebui",
                },
                json={
                    "jsonrpc": "2.0",
                    "method": "tools/call",
                    "params": {"name": tool, "arguments": args},
                    "id": 1,
                },
            )
            data = resp.json()
            if "error" in data:
                return f"Error: {data['error']['message']}"
            result = data.get("result", {})
            if isinstance(result, dict) and "text" in result:
                return result["text"]
            return json.dumps(result, indent=2)
```

**Key points:**
- Option A (native MCP) is the easier and recommended approach.
- Option B (custom tool functions) gives you fine-grained control over which tools are exposed.
- All tools are namespaced by server name to avoid conflicts.

## Knowledge Base (RAG)

Open WebUI can search your documents when answering questions. RAG stands for Retrieval-Augmented Generation. It works like giving the AI a stack of reference material to consult before answering.

### Adding Documents

1. Go to **Workspace** then **Knowledge**.
2. Click **Create a Knowledge Collection** (for example, name it "Vault").
3. Upload markdown files from your Obsidian vault (`~/Vault/`).
4. Open WebUI chunks, embeds, and indexes them automatically.

### Using Knowledge in Chat

1. Start a new chat.
2. Click the **+** button next to the message input.
3. Select your knowledge collection.
4. Ask your question. The model retrieves relevant document chunks before generating its answer.

**Key points:**
- Upload your documents once and Open WebUI indexes them automatically.
- Select a knowledge collection in chat to give the AI access to those documents.
- RAG improves answer accuracy by grounding responses in your own content.

## Prompt Enhancement

Open WebUI has automatic prompt enhancement enabled by default. When you send a message, PromptHub rewrites your prompt to be clearer and more specific before forwarding it to Ollama. Think of it as a writing assistant that polishes your question before the AI reads it.

### Configuration Details

| Setting         | Value                          |
| --------------- | ------------------------------ |
| Model           | `gemma3:4b`                    |
| Privacy level   | `local_only` (never leaves your Mac) |
| API key         | `sk-prompthub-openwebui-001`   |
| Enhancement     | Enabled (`"enhance": true`)    |

Enhancement is configured in two places:

- **`configs/api-keys.json`** -- the API key entry has `"enhance": true`, which tells the `/v1/chat/completions` proxy to rewrite prompts before sending them to Ollama.
- **`configs/enhancement-rules.json`** -- the `open-webui` client section defines the model, system prompt, temperature, and privacy level.

### Disabling Enhancement

To turn off enhancement, set `"enhance": false` for the `sk-prompthub-openwebui-001` key in `configs/api-keys.json` and restart the router.

**Key points:**
- Enhancement is on by default for Open WebUI.
- All enhancement happens locally on your Mac (privacy level: `local_only`).
- Change the `enhance` flag in `configs/api-keys.json` to toggle it.

## Monitoring

All Open WebUI requests flow through PromptHub. You can monitor them in several ways.

### Where to Look

- **Dashboard:** http://localhost:9090/dashboard (activity feed, circuit breakers, Ollama status, Open WebUI panel)
- **Activity log:** `GET http://localhost:9090/audit/activity?client_id=openwebui`
- **Health check:** `curl http://localhost:9090/health`

### Dashboard Panel

The PromptHub dashboard includes a dedicated Open WebUI panel that shows:

- **Connection status** -- a green or red badge showing whether Open WebUI is responding on its configured port (default 3000).
- **API base URL** and **MCP endpoint** -- the URLs Open WebUI uses to reach PromptHub.
- **Port** -- the port Open WebUI is running on.
- **Recent request count** -- how many requests Open WebUI has sent recently.
- **Quick link** -- a button to open Open WebUI in your browser.

The panel auto-refreshes every 15 seconds.

### Configuration API Endpoint

You can also fetch connection settings as JSON from the router:

```bash
curl http://localhost:9090/configs/open-webui
```

This returns the same structure as `python -m cli generate open-webui`. It is useful for scripting or verifying the running configuration.

**Key points:**
- The dashboard at `localhost:9090/dashboard` gives you a live view of Open WebUI activity.
- Use the activity log API to query request history for the `openwebui` client.
- The config endpoint lets you verify connection settings programmatically.

## Troubleshooting

### No models showing up

1. Verify PromptHub is running:
   ```bash
   curl http://localhost:9090/health
   ```
2. Verify the API key works:
   ```bash
   curl -H "Authorization: Bearer sk-prompthub-openwebui-001" \
     http://localhost:9090/v1/models
   ```
3. If the key returns 401, reload the keys:
   ```bash
   curl -X POST http://localhost:9090/v1/api-keys/reload
   ```

### Chat returns errors

1. Check if Ollama is running:
   ```bash
   curl http://localhost:11434/api/tags
   ```
2. Check PromptHub circuit breakers on the dashboard:
   ```bash
   open http://localhost:9090/dashboard
   ```

### Port 3000 already in use

1. Find what is using port 3000:
   ```bash
   lsof -ti tcp:3000
   ```
2. Stop that process, or use a different port:
   ```bash
   uvx --python 3.11 "open-webui>=0.8,<0.8.10" serve --port 3001
   ```

### Version update available but broken

Open WebUI v0.8.10 has a broken dependency (`ddgs==9.11.2`). Stay on `<0.8.10` until the issue is fixed upstream.

To check if a newer version works:

```bash
uvx --python 3.11 open-webui@latest serve --port 3000
```

If it fails with a dependency error, go back to the pinned version.

**Key points:**
- Start troubleshooting by checking that PromptHub and Ollama are both running.
- Reload API keys if you get a 401 error.
- Avoid Open WebUI v0.8.10 due to a known broken dependency.
