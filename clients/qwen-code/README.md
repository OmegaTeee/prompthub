# Qwen Code

Repo-managed source of truth for Qwen Code settings.

## Config path

Primary tracked files:

```text
~/.local/share/prompthub/clients/qwen-code/settings.json
~/.local/share/prompthub/clients/qwen-code/settings.direct.json
~/.local/share/prompthub/clients/qwen-code/settings.router.json
```

Primary live Qwen Code path:

```text
~/.qwen/settings.json
```

Compatibility path:

```text
~/.config/qwen-code/settings.json
```

Recommended install strategy: keep both live paths as symlinks to the active
tracked config in this directory.

## Setup

Qwen Code supports two repo-managed modes:

1. Direct LM Studio

    ```zsh
    export LM_API_TOKEN="sk-lm-..."
    ./clients/qwen-code/setup.sh direct
    ```

2. PromptHub router

    ```zsh
    export PROMPTHUB_QWEN_CODE_TOKEN="sk-prompthub-qwen-code-001"
    ./clients/qwen-code/setup.sh router
    ```

The active `settings.json` symlink is switched to the selected mode by
`setup.sh`.

## Files

- `settings.direct.json` - Qwen Code talks directly to LM Studio
- `settings.router.json` - Qwen Code talks to PromptHub `/v1`
- `settings.json` - Active tracked symlink selected by `setup.sh`
- `mcp.json` - MCP server configuration (PromptHub bridge entry)
- `setup.sh` - Recreates the tracked/live symlinks for `direct` or `router`
- `check.sh` - Prints the active mode, symlink targets, env key, model, and base URL
- `uninstall.sh` - Removes the tracked/live symlinks
- `qwen-cli-help.md` - Reference notes for the Qwen CLI
- `mcp-oauth-tokens.json` - Auto-generated by Qwen CLI; empty by default

## Notes

- `envKey` must be the environment variable name, not the key itself.
- `LM_API_TOKEN` is the canonical local LM Studio token name in this repo.
- `LMSTUDIO_API_KEY` can remain as a shell alias for backward compatibility.
- `PROMPTHUB_QWEN_CODE_TOKEN` is the dedicated PromptHub router token for Qwen
  Code mode.
- `splitToolMedia: true` is enabled for stricter OpenAI-compatible local
  servers such as LM Studio.
- If you change the model id in LM Studio, update both `model.name` and the
  matching provider `id`.

## Verify

```bash
readlink ~/.local/share/prompthub/clients/qwen-code/settings.json
ls -l ~/.qwen/settings.json

```
