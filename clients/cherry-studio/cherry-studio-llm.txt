## What is Cherry Studio?

Cherry Studio is an all-in-one AI assistant platform integrating multi-model conversations,
knowledge base management, AI painting, translation, and more. It is an Electron-based
desktop app for macOS, Windows, and Linux.

Key capabilities relevant to PromptHub integration:
- Multi-provider model aggregation (OpenAI, Gemini, Anthropic, Azure, custom OpenAI-compatible)
- MCP server support (stdio transport, built-in and external servers)
- Cherry Agent mode (Anthropic-endpoint agents with MCP tool access)
- Local knowledge base (embedding + RAG with PDF, DOCX, MD, URL, sitemap sources)
- Built-in memory system (local knowledge graph for cross-conversation persistence)

## Provider Configuration

Cherry Studio manages providers in Settings > Model Services. Each provider has:
- **API Key**: Supports multi-key rotation (comma-separated: `sk-xxx1,sk-xxx2,sk-xxx3`)
- **API Address (Base URL)**: Root URL only — Cherry Studio appends `/v1/chat/completions`
  - Trailing `/` → only `chat/completions` is appended
  - Trailing `#` → no appending (use the exact URL)
- **Models**: Click "Manage" to auto-retrieve, or manually add model IDs
- **Enable toggle**: Provider must be enabled (top-right switch) for models to appear

### PromptHub as a Custom Provider

To connect Cherry Studio to PromptHub Router as an OpenAI-compatible provider:
1. Settings > Model Services > click "+ Add"
2. Provider Name: `PromptHub Router`
3. Provider Type: `OpenAI` (or `OpenAI Response` for /v1/responses)
4. API Key: `sk-prompthub-cherry-001`
5. API Address: `http://127.0.0.1:9090`
6. Enable the provider toggle
7. Click "Manage" to discover models from the router

Note: Cherry Studio supports the OpenAI Responses API format (`/v1/responses`).
When using provider type "openai-response", the responses endpoint is used instead
of chat completions.

### Ollama / Local LLM Provider

Cherry Studio has built-in Ollama support:
- API Address: `http://localhost:11434/`
- No API key required
- Keep Alive Time: session duration in minutes before auto-disconnect
- Models: manually add downloaded model names (e.g., `llama3.2`)

The same pattern works for LM Studio or any OpenAI-compatible local server.

## MCP Server Configuration

Cherry Studio supports MCP (Model Context Protocol) servers via stdio transport.

### Environment Prerequisites

Cherry Studio uses its own bundled `uv` and `bun` (does NOT reuse system installs).
Install via: Settings > MCP Server > click "Install" button.

Executable directory:
- macOS/Linux: `~/.cherrystudio/bin/`
- Windows: `C:\Users\<username>\.cherrystudio\bin\`

If auto-install fails, manually symlink or download:
- Bun: https://github.com/oven-sh/bun/releases
- UV: https://github.com/astral-sh/uv/releases

### Adding an MCP Server

1. Settings > MCP Server > "Add Server"
2. Fill in:
   - **Name**: display name (e.g., `prompthub-bridge`)
   - **Type**: `Standard Input/Output (stdio)`
   - **Command**: executable (e.g., `node`, `npx`, `uvx`)
   - **Arguments**: args to the command (one per line or space-separated)
   - **Environment Variables**: KEY=VALUE format, one per line
   - **Long Running Mode**: toggle ON for persistent servers
   - **Timeout**: seconds (default 60)
3. Advanced Settings (optional):
   - Provider: author/org name
   - Provider URL, Logo URL, Tags
4. Click "Save"

### JSON Configuration Format

Cherry Studio uses `mcpServers` format (same as Claude Desktop). Accessible via
the "Edit JSON" button in Settings > MCP Server. Example:

```json
{
  "mcpServers": {
    "prompthub-bridge": {
      "isActive": true,
      "name": "prompthub-bridge",
      "type": "stdio",
      "description": "local router",
      "baseUrl": "",
      "command": "node",
      "args": [
        "/Users/<you>/prompthub/mcps/prompthub-bridge.js"
      ],
      "env": {
        "PROMPTHUB_URL": "http://127.0.0.1:9090",
        "AUTHORIZATION": "Bearer sk-prompthub-cherry-001",
        "CLIENT_NAME": "cherry-studio",
        "SERVERS": "memory,context7,sequential-thinking,desktop-commander,perplexity-comet"
      },
      "longRunning": true,
      "provider": "OmegaTeee",
      "installSource": "manual",
      "isTrusted": true
    }
  }
}
```

Cherry-specific fields beyond standard mcpServers:
- `isActive` (bool): whether the server is enabled
- `name` (string): display name in the GUI
- `type` (string): transport type, always "stdio" for PromptHub
- `baseUrl` (string): empty for stdio servers
- `longRunning` (bool): keeps the server process alive between calls
- `installSource` (string): "manual" for hand-configured servers
- `isTrusted` / `trustedAt` / `installedAt`: trust and audit metadata
- `provider` / `providerUrl` / `logoUrl` / `tags`: GUI display metadata

### Using MCP in Chat

After adding an MCP server, enable it per-conversation:
1. Open a chat topic
2. Click the MCP Server button in the chatbox toolbar
3. Select which MCP servers to activate for this conversation
4. The model will have access to the server's tools

### Built-in MCP Servers

Cherry Studio includes several built-in MCP servers (prefix `@cherry/`):
- `@cherry/memory` — persistent knowledge graph memory (set MEMORY_FILE_PATH)
- `@cherry/sequentialthinking` — structured step-by-step reasoning
- `@cherry/brave-search` — web search via Brave API (set BRAVE_API_KEY)
- `@cherry/fetch` — fetch web content from URLs
- `@cherry/filesystem` — local file system operations
- `@cherry/mcp-auto-install` — auto-install MCP services (beta)

### MCP Troubleshooting

Common issues:
- Server not starting: check that uv/bun are installed in `~/.cherrystudio/bin/`
- Connection timeout: increase Timeout setting or enable Long Running Mode
- Tools not appearing: ensure the server is enabled AND selected in the chat toolbar

## Cherry Agent

Cherry Studio v1.7.0+ supports Agent mode — autonomous agents with tool/MCP access.

Setup:
1. Create an Anthropic-type provider (any Anthropic-compatible endpoint)
2. Enable API Server in settings
3. Create an Agent — right-click to configure:
   - Permissions (file access, etc.)
   - Available tools and MCP services
   - Model selection (plan model, small model)

Agents can use MCP servers for tool calling, making them compatible with PromptHub's
bridge for full tool access.

Note: Agent mode consumes many tokens — monitor usage carefully.

## Knowledge Base

Cherry Studio supports local RAG (Retrieval Augmented Generation):

### Setup
1. Add an embedding model (e.g., `bge-m3`) via Model Services > filter "Embedding Model"
2. Create knowledge base: Knowledge Base panel > "Add" > name + select embedding model
3. Add data sources:
   - Files: PDF, DOCX, PPTX, XLSX, TXT, MD, MDX
   - Folder directory (auto-imports supported formats)
   - Website URL
   - Sitemap XML
   - Plain text notes
4. System auto-vectorizes — wait for green checkmark

### Using in Chat
1. Create/open a topic
2. Click "Knowledge Base" in the chatbox toolbar
3. Select which knowledge base(s) to cite
4. Ask questions — model generates answers from retrieval results
5. Cited sources are attached below the answer

### Limitations
- Images in documents are not vectorized (convert to text manually)
- Website crawling may fail on sites with anti-crawling measures
- Most sites provide sitemaps at `/sitemap.xml`

## Chat Interface

### Assistants and Topics
- **Assistant**: personalized model settings (prompt, parameters, default model)
- **Topic**: a conversation under an assistant; shares the assistant's settings
- Multiple topics can exist under one assistant

### Key Model Parameters
- **Temperature** (default 0.7): 0-0.3 = deterministic, 0.4-0.7 = balanced, 0.8-1.0 = creative
- **Top P** (default 1): nucleus sampling threshold for vocabulary diversity
- **Context Window**: number of messages retained (5-10 for general, >10 for complex tasks)
- **MaxTokens**: max response length (500-800 chat, 2000-3600 code, 4000+ long text)
- **Stream Output**: typewriter effect (on by default; turn off for models like o1-mini)
- **Custom Parameters**: additional request body fields (presence_penalty, etc.)

### Chatbox Toolbar Buttons
- New Topic, Upload Image/Document, Web Search, Knowledge Base
- MCP Server toggle, Generate Image, Select Model
- Quick Phrases, Clear Messages, Expand, Clear Context
- Estimated Token Count, Translate

## Data and Storage

### Config Persistence
All Cherry Studio settings (providers, MCP servers, assistants, memory) are stored in
Electron's LevelDB under a single `persist:cherry-studio` key:
- macOS: `~/Library/Application Support/CherryStudio/Local Storage/leveldb/`
- NOT in an external config file — use the GUI or Edit JSON dialog

The `config.json` in Application Support only contains Electron prefs (theme, tray).

### Other Data Paths
- `~/Library/Application Support/CherryStudio/Data/agents.db` — agents and sessions (SQLite)
- `~/Library/Application Support/CherryStudio/Data/Memory/memories.db` — built-in memory
- `~/Library/Application Support/CherryStudio/Data/KnowledgeBase/` — RAG embeddings
- `~/Library/Application Support/CherryStudio/Data/Files/` — uploaded documents
- `~/.cherrystudio/bin/` — bundled uv and bun executables

### Backup
- Local backup, WebDAV backup, and scheduled backups supported
- Settings > Data Settings for configuration
